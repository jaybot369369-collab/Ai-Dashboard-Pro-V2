## How theming works (read this first)

All colors, radii, shadows, and font tokens live in **CSS custom properties** on `:root` in `css/styles.css`.

Existing themes are layered like this:

```css
:root                   { --bg: #0d1117; --accent: #4f8ef7; ... }   /* dark (default) */
[data-theme="light"]    { --bg: #f0f2f8; --accent: #3b74e8; ... }   /* light          */
```

The active theme is set on `<html data-theme="...">` (toggled in `js/app.js → applyTheme()`).

**Therefore every new skin = one new `[data-theme="..."]` block + (optional) extra rules** — never edit existing tokens, never hardcode colors elsewhere, never break the light/dark themes.

## Files you may touch

| File | Purpose |
|------|---------|
| `css/styles.css` | Add the new `[data-theme="<name>"]` block at the bottom, plus any skin-specific overrides (gradients, shadows, font swaps) |
| `js/app.js` | If adding a new selector UI: extend the existing theme toggle — don't rip it out |
| `index.html` | Only if loading new fonts (`<link rel="preconnect">` + Google Fonts) |

## Files you must NOT touch

- Anything in `js/tabs/`, `js/integrations/`, `js/data.js`, `js/charts.js` — that's logic, not skin
- Module IIFE patterns — leave `const FooTab = (() => { ... })();` alone
- localStorage `jb_*` keys
- `js/data/daily_report.json` (auto-generated by cron)
